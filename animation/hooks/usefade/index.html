useFade

This animation perform a fade to the component, you can specify if it will be a fade in, fade out or both.

You can also control the render with the animation state property.

Options object:

useFade receives an object for custom configurations, like:

propertydescriptiontypedefault value
startOnRenderstart fadeIn on first component renderbooltrue
durationthe duration of the fade effect, it is a ms valuenumber300
fadeInapply the fade in animation when the animation state is truebooltrue
fadeOutapply the fade out animation when the animation state is falsebooltrue

Returned values:

useFade hook return a pair of values, the animation object and a function to toggle the animation state.

animation properties:

propertydescription
animationanimation css property value
stateanimation state value (after fade ends)
instantStateanimation state value that updates right before it changes (without waiting the fade to end)

toggle function:

Second value returned from useFade hook is a function to toggle the fade animation, the initial state value is true (if you didn’t change the startOnRender property) and it will perform a fade in animation by default.

If you want to change the animation state to false to perform a fade out, just call the function passing false as an argument.

Example:

fade.jsx
function App() {
  const [{ animation, state, instantState }, toggleFade] = useFade({
    startOnRender: true,
  });


  function handleClick() {
    toggleFade(!state);
  }


  return (
    <Wrapper height="200px">
      <div>
        <p>
          state: <code>{state.toString()}</code>
        </p>
        <p>
          instantState: <code>{instantState.toString()}</code>
        </p>
      </div>


      <Button skin={instantState ? 'danger' : 'success'} onClick={handleClick}>
        Toggle card
      </Button>


      {state && (
        <Card animation={animation}>
          <Card.Content>Frigobar card</Card.Content>
        </Card>
      )}
    </Wrapper>
  );
}


render(App);

state: true

instantState: true

Take a look that we are using the state property to control the render and make sure that our Card component only unmounts when fadeOut finishes.
